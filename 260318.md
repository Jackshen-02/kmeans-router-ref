# Progress Update

### What I Implemented
- PyTorch K-Means reference for key clustering in the forward pass only
- BF16 input support with FP32 internal accumulation for distance computation and centroid updates
- Naive and vectorized implementations for correctness checking
- Cluster-based reordering/scatter logic for downstream sparse attention routing
- Code repository: [PyTorch BF16 K-Means reference](https://github.com/YOUR_USERNAME/kmeans-router-ref)

### Observations
- Naive and vectorized implementations matched in all tested CPU cases.
- Objective was monotonic non-increasing in every run.
- BF16 runs were stable under the reference mixed-precision path; no naive/vectorized divergence was observed.
- The vectorized path showed clear CPU speedups over the naive baseline; observed speedups ranged from `2.8x` to `245.5x` depending on size and dtype.
